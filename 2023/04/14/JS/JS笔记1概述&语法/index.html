JS笔记1 概述&语法

JS基础知识

浏览器浏览页面的过程

浏览器的功能：下载HTML，解析HTML，下载CSS，解析CSS，渲染界面，下载JS，解析JS，执行JS。

功能模块：用户界面、渲染引擎、JS引擎、存储等。

一个浏览器会产生多个进程，一个进程会产生多个线程，而这些功能模块一般各处于不同的线程当中。

当浏览器打开一个页面时，会开启一个进程，包括了一个渲染引擎线程，一个JS引擎线程。

JS是单线程的，JS改变渲染效果是通过跨线程通信来实现的。

栈和堆内存分配

栈Stack和堆Heap用来存数据，变量名单独存一个区。

Stack区特点：每个数据顺序存放。

Heap区特点：每个数据随机存放。

JS里每个数字占64位。

JS不是对象的有：数字，字符串，布尔，其他都是对象，如数组、函数。

非对象存在Stack，对象都存在Heap。

当声明一个变量，并赋值一个对象时，变量名存在变量名区，对象存在Heap区，然后变量名对应的Stack区存的是对象在Heap中的地址。（当赋值的是一个非对象时，变量名对应的Stack区存的是非对象的值，是一个数据。）

=号就是直接复制，右边指向的是地址，就复制地址，右边指向的是数据，就复制数据（所以并没有传值和传址之分）。

细节

window变量和window对象是两个东西，window变量是一个容器，存放了window对象的地址，window对象是堆Heap里的一堆数据。

如果定义var a = window，那么a将起到和window变量一样的作用。（但是最好不要这么做，因为会混淆）

同理console和console对象也是不同的。（console是window对象里的一个属性）

prototype原型链

JS初始就在内存里存放了window.object.prototype对象和window.array.prototype对象。

var bj={}; obj.toString();

上述代码并不会报错，因为定义一个object对象时，其在Heap中的区域会包含一个隐藏属性，存的是window.object.prototype的地址。当obj.toString()发现在obj中没有toString()时，就会去隐藏属性中找。prototype中包含了toString()，因此name调用toString()就会调用prototype中的toString()，而不会报错。

注：如果写上 name.toString()=”xxxx”;并不会影响 var c={};c.toString();的效果，因为name.toString()=”xxxx”会在name的Heap区中产生一个toString(),并不会越级影响到prototype中的toString。除非直接修改window.object.prototype.toString() 。

所以，prototype里存的是所有普通对象所通用的共同属性。方便复用共有属性。

注：每个对象都有一个隐藏属性，指向原型。

我们此处只关心小写字母对象的隐藏属性，不展开了解大写首字母对象的隐藏属性（因为较复杂）。

与上同理，数组对象也可直接调用arr.join(‘-‘)而不报错。

canvas

canvas标签有点类似于image标签，具有HTML意义上的宽高属性，canvas的宽高在一开始定义的时就已经确定了，在CSS中再去指定它的宽高，实际上是缩放，可能会导致图像模糊。（应该在JS中使用canvas对象.width=n px的方式来修改）（注：vh和vw是CSS单位，HTML没有这个单位）

语法

表达式，语句，值，返回值的区别。

语句是可以单独执行的、能够产生实际效果的代码；而表达式则是包含在语句中，根据某种条件计算出一个值或得出某种结果，然后由语句去判断和处理的代码。

表达式有值，而语句的值是undefined或者返回值。

语句有：声明式语句，赋值语句，执行式语句（使用整个语句块来执行某个动作，如条件语句，循环语句）。

JS中是区分大小写的。

大部分空格是没有实际意义的，只要不影响断句。

回车也是一样。（例外：return后面加回车会改变原意，return后面加回车，会自动补充一个undefined）

变量名的第一个字符不能是数字，其他基本都可以（甚至是希腊字母）。

&&短路逻辑

如果&&前面的为真值，那JS就会执行后面的代码。

如果&&前面的为假，JS就不执行后面的代码。

拓展：A && B && C

||逻辑

A为真值，则不执行后面代码。

A为假，则执行后面的。

拓展：A||B||C||D 取第一个真值或者D

总结条件语句

  1. if else 最常用，但是代码简短的时候，还是简写最好。

  2. switch 结构简单，但要用的话，一定不要少了break

  3. A?B:C 非常常用

  4. A&&B 非常常用（ 如fn&&fn() 如果fn存在则调用fn() ）

  5. A||B 也很常用（ 如A=A||B 如果A存在则A等于B）

循环语句

while循环

还是for循环更常用。

语法：while(表达式){语句}

  1. 当表达式为真时，执行后面的语句。
  2. 执行完毕后再次回来判断表达式真假。
  3. 当为假时，跳出while循环。

其他还有do while循环。

for循环

for(语句1;表达式2;语句3){循环体}

  1. 先执行语句1(初始化)。
  2. 然后判断表达式2。
  3. 如果为真,执行循环体。
  4. 循环体之后执行语句3。
  5. 然后再判断表达式。
  6. 如果为假,退出for循环。

for循环如果不写中间的判断条件表达式2的话，就会进入死循环。
