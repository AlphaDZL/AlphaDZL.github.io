CSS定位

布局是屏幕平面上的，定位是垂直于屏幕的。

背景background是由border外边距围成的区域组成。

div的分层

（层叠上下文）

一个div的分层，从下往上依次是background，border，块级子元素，浮动元素，内联子元素。

文字最高，div在文字的下面

div文字内容是盖住浮动元素里的文字内容的

如果是正常的文字，与文字在哪里无关，按照出现的先后顺序来覆盖。

position的五个取值

position: static;

默认的值，当前元素在文档流中。（这个值一般不用自己写出，默认就是）

position: relative;

还是在文档流当中，但是可以有一定的偏移。占的实际位置不变，只是显示的地方与原来有一定偏移。

一般用来做absolute元素的爸爸元素。

一般与z-index搭配使用

默认每一个元素的z-index是auto，auto计算出来的值是0，但是与直接写0还是有区别的。

z-index值越大，越优先显示在上层。（z-index值可以为负）

position: absolute;

一般会有一个爸爸元素，并且爸爸元素会有position: relative;

直接相对于爸爸元素偏移后的位置。

会找祖先里最近的那一个定位元素（只要不是position: static;就是定位元素）进行相对定位，不一定是直接的上一层元素。

使用场景：脱离原来的位置，另起一层，比如对话框的关闭按钮。以及鼠标提示。

tips:

white-space: nowrap;

（文字内容不准换行）

也可以搭配z-index使用。

  • 有些浏览器如果不写top/left会位置错乱，因此最好写上，即使top/left值为0。
  • 善用left:100%;
  • 善用left:50%;加负margin

position: fixed;

相对于视口定位，但是如果fixed元素所在的容器，container有一些特殊的属性，那么它可能就不是相对于视口定位的。

不要把一个fixed元素放到一个具有transform属性的容器中。

使用场景：广告，回到顶部按钮。

配合z-index使用。

不要在手机端用position: fixed;会出现很多问题

position: sticky;

粘滞定位，一般用来做导航。（兼容性较差）

总结

·position

  • static默认值，待在文档流里

  • relative相对定位，升起来，但不脱离文档流

  • absolute绝对定位，定位基准是祖先里的非static

  • fixed固定定位，定位基准是viewport(有诈)

  • sticky粘滞定位，不好描述直接举例

  • 经验

  • 如果你写了absolute,一般都得补一个relative

  • 如果你写了absolute或fixed,一定要补top和left

  • sticky兼容性很差，主要用于面试装逼

z-index如果是负值，则会比background还要低。（不会逃离自己所在的层叠上下文）

层叠上下文

  • 每个层叠上下文就是一个新的小世界（作用域）
  • 这个小世界里面的z-index跟外界无关
  • 处在同一个小世界的z-index才能比较

会创建层叠上下文的情况

  1. z-index:0;
  2. 父元素display:flex;
  3. opacity不为1
  4. 有transform属性
  5. 常用的为以上几种，详细搜索“层叠上下文 MDN”

负的z-index仍在小世界里。只能逃出没有层叠上下文的那个div（-1不一定能逃出一个div，除非这个div不是一个层叠上下文）
