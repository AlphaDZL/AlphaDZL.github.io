CSS动画

CSS动画的原理是每过一段时间，将div移动一小段距离，知道将div移动到目标所在位置。

做动画的方式

改变left来做动画

不停的改变div的left来形成动画效果。（整个过程不停重新渲染，因此性能不好）

使用transform语法做动画

性能更好

浏览器渲染过程

  1. 根据HTML构建HTML树(DOM)
  2. 根据CSS构建CSS树(CSSOM)
  3. 将两棵树合并成一颗渲染树(render tree)
  4. Layout布局（文档流、盒模型、计算大小和位置）
  5. Paint绘制（把边框颜色、文字颜色、阴影等画出来）
  6. Composite合成（根据层叠关系展示画面）

如何更新样式

一般使用JS来更新样式（用户鼠标hover之类的也可以，但大部分还是用JS）

div.style.background=’red’

div.style.display=’none’

div.classList.add(‘red’)

比起上面的单条加样式，不如直接加类，因此一个类里面可以有很多的样式

div.remove()

直接删除节点

使用JS来更新动画经过的步骤

有三种更新方式

JS/CSS>样式>布局>绘制>合成

比如div.remove(),删除了一个元素，后面的元素会往前重新排，因此会重新布局和绘制。

JS/CSS>样式>绘制>合成

布局没变，比如仅改变背景颜色之类的，因此不需要重新布局。

JS/CSS>样式>合成

没改变布局，也没改变颜色，因此只需要合成即可。比如transform只合成。

正确的做动画的方式

不要用改变left的方式做动画，因为用left做动画，每一次变化left都会重新渲染，导致性能不好。

正确的做法是使用transform（变形）

transform: translateX(0=>300px)

transition过渡属性可以自动脑补中间帧。

特点

  • 没有relayout（重新布局）和repaint（重新绘制）

  • 比改left性能好

transform

CSS动画优化

  • 用transform做动画，代替用left做动画。
  • 使用will-change或translate。
  • 详情参考谷歌官方文章。

四个常用功能

位移translate

transform: translateX(n px);

在X方向上移动n像素。当它变大，就往右，当它变小，就往左。

transform: translateY(n px);

当Y方向上移动n像素。当它变大，就往下，当他变小，就往上。

transform: translateZ(n px);

垂直与屏幕的方向。

perspective: 1000px;（父元素上）

透视图的视点在视口正中心，屏幕里面1000px远

translateZ越大，图越大，translateZ越小，图越小，且越靠近视点。

tips

  1. transform: translateX(50%); 往右偏自己宽度的一般。
  2. 绝对定位元素的居中（经常用到）
  3. translate的XYZ可以缩写。

缩放scale

translate: scale(1.5);

变大50%

translate: scaleX(1.5);

变胖50%

translate: scaleY(1.5);

变高50%

translate: scale(1.5,1.5);

XY同时变化

tips

  1. scale变化的时候border也会一起变化
  2. scale用的比较少，因为容易出现模糊。

旋转rotate

transform: rotate(45deg);

顺时针方向旋转45°。（以垂直于屏幕的轴转动）

transform: rotateZ(45deg);

与第一条（不写Z）效果一样。

transform: rotateX(45deg);

以X轴为转轴，转动45°。（用的少）

transform: rotateY(45deg);

与第三条同理。

其他还有rotate3d，3d上转动，较复杂，参考文档。

倾斜skew

这种转换是一种剪切映射 (横切)，它在水平和垂直方向上将单元内的每个点扭曲一定的角度。每个点的坐标根据指定的角度以及到原点的距离，进行成比例的值调整；因此，一个点离原点越远，其增加的值就越大。

用的少，参考文档。

tips

  1. 如果想要动画效果可以加上transition: all 1s;
    (第三个参数为时间一秒)
  2. 当做一个hover时候的动画时，hover的时候会产生一个动画效果，但当鼠标没有再hover的时候，会发生一个与原来反向的动画，是元素回到原来的位置。
  3. transform可以同时使用多种方法，只需要用空格隔开即可。

transition

用作：添加中间帧。

语法

transition:属性名 时长 过渡方式 延迟

在变化前的类中写上。

过渡方式

过渡方式 默认为ease自然方式，不会显得突兀。如果改成线性 linear，就会直接以一定速度变化，到位置后直接停止，显得有点突兀。

过渡方式有：linear|ease|ease-in|ease-out|ease-in-out

其他的还有cubic-bezier|step-start|step-end|ends等，具体参考MDN，里面介绍的很明了。

延迟

过多久开始动画。（很少用到）

注意

不是所有的属性都能过渡，需要注意的情况有：

  1. display: blcok=>none,一般改成 visibility: visible=>hidden（后者不见了，但是还占位置，详细的情况，查询display和visibility的区别）
  2. background颜色是可以过渡的。
  3. 不建议让opacity透明度过渡。
  4. 过渡必须要有开始和结尾。
  5. 一般只有一次或者两次动画（比如hover和非hover状态的过渡）。

animation

语法

  1. 申明关键帧keyframes
  2. 添加动画

keyframes

两种语法

第一种，使用百分比（使用更加广泛一点）。

第二种，from to

animation:

animation: 时长|过渡方式|延迟|次数|方向|填充模式|是否暂停|动画名;

直接在变化后的类中写上即可。

时长：1s或者1000ms。

过渡方式：linear|ease|ease-in|ease-out|ease-in-out等，与transition取值一样。

延迟：动画应在指定的时间量过去后开始。默认值为 0s，表示动画应立即开始。

次数：动画重复的次数，3或者2.4或者infinite。默认为1。

方向：normal|reverse|alternate|alternate-reverse。

normal：默认的，一般不写即可。

reverse：每个循环中反向播放。

alternate：每个循环中正反交替播放，第一次迭代是正向播放。

alternate-reverse：alternate-reverse。

填充模式：none|forwards|backwards|both

none：动画未执行时，动画将不会将任何样式应用于目标。（默认值）

forwards：保留执行期间遇到的最后一个关键帧。

backwards：动画将在应用于目标时立即应用第一个关键帧中定义的值，并在animation-delay期间保留此值。

both：同时应用forwards和backwards。

是否暂停：paused|running

一般与JS搭配用。

动画名：自己缩写的keyframes的名字。
